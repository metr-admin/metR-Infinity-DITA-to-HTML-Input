Administration

The Mind Administration module enables you to configure and manage various functionalities that apply to the entire Mind application.

Using the Administration module, you can perform security management, user management, reports management, and manage custom forms and workflows.

Roles in Administration

There are two types of default admin roles in Mind:

  • Customer Administrator
  • Mind System Administrator

The Customer Administrator and the Mind System Administrator have different field-level permissions assigned to their respective roles. The Customer Administrator and the Mind System Administrator share control over the administration module.

The customer manages all project and form-level controls.

The Mind System Administrator has additional control to:​

  • Configure application-wide settings
  • Manage other critical controls for the customer
  • Workflow Management - Mind can help create workflows for the customer without accessing the customer database

The Mind System Administrator manages all critical and technical application-level controls. During implementation, the Mind system administrator configures the application as a one-time setting.

The Mind System Admin should be able to assist the customer if required without accessing the customer's Mind application. The Mind system administrator retains control over specific configurations to debug the system.​
