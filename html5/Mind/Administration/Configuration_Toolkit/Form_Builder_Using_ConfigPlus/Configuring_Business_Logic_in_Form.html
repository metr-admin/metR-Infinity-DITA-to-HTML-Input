Configuring Business Logic in a Form

This section explains the procedure to configure business logic in forms using the FORM BUSINESS LOGIC tab. The example fields and options selected in the procedure are for instructional purposes.

Business Scenario Used

In an Expense form, if Direct Expenses is selected, it has a Purchase Order No associated to it. Once the Purchase Order No is selected, you want to update the Total PO Cost for the associated Purchase Order.

  1. In the Module menu, click Administration.
    The ADMINISTRATION page is displayed.
  2. In the navigation pane, expand theConfiguration Toolkitfolder, and then clickForm Builder.
    TheFORMS LISTpage is displayed.
  3. Click Import XML.
    The IMPORT FORM page is displayed.
  4. To import the Expenses form, perform the following steps:
    1. Select Form Import.
    2. In the Select Form box, search for Expenses form, and select it from the drop-down list.
    3. Click Import.
      A form with the same Form ID and Form Header of the Expenses form is created and displayed in the FORMS LIST page.
  5. To modify and update the Expenses form, select the form, and click Edit.
  6. In the controls pane, click SIMPLE INPUT CONTROLS, and then drag and drop Numeric control to the design area.
  7. Double-click the Numeric control and enter Total PO Cost for the text box.
  8. To make the FORM BUSINESS LOGIC tab available, in the toolbar, click Switch to Advanced.
  9. Click the FORM BUSINESS LOGIC tab.
  10. In the controls pane, from the Select Event drop-down list, select the Form Load event.
  11. Expand the DEFINITIONS block and click the Variables block.
  12. Click Create Variable.
    A pop-up message appears to provide the variable name.
  13. In the New variable name field, enter the variable name as POValue, and click OK.
    The newly created variable is available in the variable list with the set operator.
  14. Drag and connect the set operator to the Form Load event.
  15. Expand the DATA OPERATIONS block and click the Read Data block.
    The available options of the Read Data block are displayed.
  16. Drag and connect the SELECT block to the set operator.
  17. In the SELECT block, select the Module as Project, Form as Purchase Order, and Field as Revised PO Value in {_LOC:LOC_CURRENCY_SYMBOL}:.
  18. In the WHERE block, select the Module as Project, Form as Purchase Order, and Field as PR Ref Number.
  19. To equate the conditions of the SELECT and WHERE blocks, click the GET block under the DATA OPERATIONS block and drag and connect the Get Value of field to the WHERE block.
  20. Select the value of the Get Value of field as Purchase Order No:.
  21. Expand the ACTIONS block and click Set block.
  22. Drag and connect the Set Value of field to the Form Load event.
  23. Select the value of the Set Value of field as Total PO Cost.
  24. Click the GET block under the DATA OPERATIONS block, and drag and connect the Get Property of to the Set Value of field.
  25. Select the value of the Get Property of field as POValue and in the next drop-down list, select the value as PurchaseOrder. Revised PO Value in {_LOC:LOC_CURRENCY_SYMBOL}:.
  26. Click Save.
  27. Click Publish.
