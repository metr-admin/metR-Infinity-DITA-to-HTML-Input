Modify Form

The Modify Form event enables you to change the form specification before the form is generated. For example, you can modify the name of a field, whether a field should be visible or read-only, and so on. The logic is executed when you click New, Edit, View, or double-click the record, and the execution is completed before the form is generated.

Use Case# Hide a field if a user belongs to a specific role

If you want to hide the Amount field for Contractors and in a particular workflow status, you can use the Modify Form event.

Use Case# Set a tooltip based on user role or workflow status

If a single field must have different information based on the information entered by a Project Manager or a Contractor, you can use the Modify Form event to set different tooltips based on the user role or based on the workflow status. For example, Comments field.

Use Case# Set a field as read-only based on user role or workflow status

In the Request For Information (RFI) form, if you want to make the RFI question as read-only for Project Manager when the record is in Submitted workflow status, you can use the Modify Form event to set the field as read-only based on the user role and subsequent workflow status.
