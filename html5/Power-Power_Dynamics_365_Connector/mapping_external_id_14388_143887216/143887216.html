Mapping External Id

You must create a mapping for the External Id field prior to initiating an object sync. After you have completed the initial mapping, perform the following tasks in order:

  1. Create a custom field named "External Id" with field type String/Text in both the source and destination system.
  2. Create a mapping field record named "Id" for the object to be synced, using the API names specified in the previous step.
Parent topic: Importing and Exporting Field Mapping

To map the Id field for object sync

  1. From the left panel under Groups, select Map Fields.
    The Map Fields page is displayed.
  2. From the ribbon at the top of the page, click New.
    The New Map Field page is displayed.
  3. Enter "Id" in the Name field.
  4. Enter "Id" in the Display Name field.
  5. Enter "UniqueIdentifier" in the Type field.
  6. In the Dynamics Field Name field, enter the API name of the External Id field in Dynamics. For example, "apttus_externalid".
    Note:
    To avoid errors, copy-and-paste the field name from Dynamics.
  7. Enter the API name of the External Id field in Power in the Power Field Name field. For example, "Apttus_ExternalId__c".
    Note:
    To avoid errors, copy-and-paste the field name from Power.
  8. Search and select the Object Mapping Name. This is the mapping name of the field's object (for example "clm_agreement Mapping").
  9. Click Save or Save & Close.
