Replacing the Digi Edgeport Interface

Purpose

This document describes how to replace a failed Digi Edgeport model 416 DB-25, on a COMSOC 1 Interface. The procedure helps to mitigate a problem with TCP/IP communication indicated when frequent COM-SOC II error messages occur during automatic or manual Saves from the sections (not MOC Save). If the FSC shares the server with the PIC application (called a Combo system), this procedure should not be necessary because it is part of every upgrade. Consult the personnel responsible for the PIC to determine when the procedure was last performed.

Application

Space plants with COMSOC 1 Interface.

Safety

Observe Space standard safety requirements and procedures defined by the Global Environmental Health , andamp; Safety (EH, andamp;S) organization, as well as all applicable local, regional, and national requirements. Safety equipment and instructions specific to the completion of this work instruction are detailed in the Instructions Section.

Procedure

  1. In order to replace the Digi Edgeport Interface follow these steps:
    1. Label the cables plugged into the serial ports so they can be connected to the same ports on the new Edgeport.
    2. Unplug the AC power source and remove the failed Edgeport.
    3. Install the replacement Edgeport and plug in all cables, but leave the AC power source unplugged.
    4. From the SYSTEM account, delete the USB configuration files to remove the devices registered to the old Edgeport. (Leave the AC power to the Edgeport unplugged.)
      1. Log on as SYSTEM
      2. Set the default directory with the command:

        set def sys$specific:[sysexe]

      3. Delete the files with the command:

        del usb$ucm_devices.dat;*

    5. Reboot the FSC. While the FSC is restarting, plug in the AC power to the Edgeport.
    6. Test whether a timing print can be obtained from all COM-SOC/CMC connected lines. If there are failures, reboot the FSC again.
  2. Hints and Tips:
    1. If the USB configuration files are not deleted, the system will assign the serial ports to be TXD16 through TXD31. Problem #1: MOC does not recognize these ports. Problem #2: It will then be necessary to use MOC to reconfigure all of the ports being used on the old Edgeport. Therefore, the easiest method of making the exchange is to delete the previous configuration files and allow the system to generate new ones.
    2. Instructions for logging into the Integrity console and rebooting the server are provided in SOP-12-002-A , andamp; UP.
    3. The two LEDs labeled P1-8 and P9+ will periodically blink red until the firmware update from the Integrity has been downloaded. Once these lights begin periodically blinking green, the Edgeport should be functioning correctly.
    4. The AC to DC power pack voltage is 5VDC. If you plug in a 12VDC supply, the connector may fit but the Edgeport will be damaged.
