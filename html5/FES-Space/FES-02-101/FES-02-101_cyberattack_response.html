Cyberattack Response

Purpose

To define a process for all Space plants to protect the assets in the event of a cyberattack of the Forming Systems.

Application

This document applies to all Space manufacturing facilities, and shall be visible at all times in control rooms and production meeting rooms for the awareness and use by every employee involved with forming area devices.

Procedure

  1. In preparation for a potential cyberattack, follow these steps:
    1. For the Machine Server
      1. Verify that the machine server backups are occurring, using document FES-03-117 Machine Server Backups for guidance (see Section 4 for references to associated Forming Electronic Work Instructions).
      2. If the machine server backup is not activated, contact your Forming Electronics (FES) DNL
    2. Ensure the backup on other Forming Manufacturing Computers are being done properly.
  2. The following situations may be warning signs of a potential cyberattack:
    1. It is not possible to log into the control system, either at the equipment, in the plant, or remotely.
      1. This inability to log into the control system has to happen for several users trying to log into any one type of Forming System (i.e. Machine Servers, Forming Supervisory Computer, PIC/PRS, LineServer, FlexIS, UC, XPAR, Gob Weight Control systems, etc.)
    2. The login control system screen is absent even after a reboot; instead there are several tasks running in the background without the ability to stop them.
    3. Users are able to log into the control system but are not able to perform any control actions, even when using the correct login credentials.
      1. In this case, the control system screens will appear to be frozen to any user input.
    4. Users are able to log into the control system but, although the network cable is connected, they are experiencing one or more of the following problems:
      1. Inability to find data that was previously in the database
      2. Unexpected pop-up windows are shown
      3. Server is running slowly
      4. Missing storage space
      5. Missing files
      6. Unusual crashes or error messages
      7. Excessive network activity and/or slower-than-normal network response
  3. If you experience any of these problems, perform the following steps:
    1. Open an incident with the I.T. Global Service Desk to inform them of the ongoing situation, and request that Information Security verify if the system is actually being compromised by a cyberattack.
      1. Make sure to define the I.T. incident: “Cybersecurity Incident” and mark it as a “High Priority”, stating which lines and machines are being affected due to the ongoing issue.

    2. Communicate to the Immediate Supervisor and the Plant Engineer that the cyberattack has been reported and is under review.
      1. The Plant Engineer shall inform the Plant Manager, Unit Plant Managers (UPMs), Line Leader, and Job Change Coordinator for the plant.
      2. Ensure that the incident is reported to Discipline Network Leaders (DNLs) of Forming, IS Machine Maintenance, and Job Change.
  4. In general, Space recommends that the response follow the priorities listed below:
    1. Protect personnel and assets – ensure that all employees are safe, and are not executing activities that could put themselves or others at risk.
    2. Wait for instruction from I.T. for next steps.
    3. Be prepared to follow your plant emergency protocol instructions, if deemed necessary.
  5. After receiving the incident notification, Information Security will analyze the situation and assign an initial severity classification in accordance with the SIRP (Security Incident Response Plan) which activates the Space Cyber Crisis Management Plan.
    1. Further instructions from the Operations Technology Lead will follow.
    2. If the cyberattack is confirmed, the Space Cyber Crisis Management Plan will be activated
