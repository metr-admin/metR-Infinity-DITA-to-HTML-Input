Integrity Server Performance Tuning

Purpose

This document describes how to improve server performance using the AUTOGEN utility of HP OpenVMS on the Integrity Server FSC (Forming Supervisory Computer). This tuning procedure can also be performed on Alphaserver and MicroVAX servers but should not be necessary.

Application

  1. This work instruction applies to all Space Plants that has FSC and Space IS Machines.
  2. The procedure helps to mitigate a problem with TCP/IP communication indicated when frequent COM-SOC II error messages occur during automatic or manual Saves from the sections (not MOC Save).
  3. If the FSC shares the server with the PIC application (called a Combo system), this procedure should not be necessary because it is part of every upgrade. Consult the personnel responsible for the PIC to determine when the procedure was last performed.

Procedure

  1. In order to improve FSC Integrity server performance follow these steps:
    1. If the FSC is a DEC MicroVAX or Compaq Alphaserver, use the console terminal or PC terminal emulator connected to the console port. The console device is typically in the immediate vicinity of the FSC. Go to step 5 when you have identified the console device. For an Integrity, proceed to step 2.
    2. If the FSC is an HP Integrity Server, either use the console device next to the FSC, or use a terminal emulator to connect to the Integrity Server Management Port.
    3. If necessary, log on as user “Oper” with the password “console”.
    4. At the Management Port Main Menu, type “co” and press Enter to connect to the VMS Console.
    5. Test whether you have keyboard control by typing Ctrl-w. If there is a message saying that you have read-only access, perform the following three-character sequence shown in parentheses to take control (Ctrl-e, c, f). Hold the Ctrl key down only when typing the letter “e”. Type all three characters in the time it would take to say them out loud.
    6. Press the Enter key to test the response. The console could be logged off in which case you will be prompted for a Username, or the console could be logged onto the FSC menu by the person responsible for backups.
      1. It is also possible that the console might be still logged on as a privileged VMS user. If so, type “log” and press Enter to log off and then proceed to step 6.
    7. If the console was not logged in, type “fsc” at the username prompt to log in. Then navigate to the Backup Menu and perform the appropriate backup within the capabilities of the FSC server model. If the FSC is an HP Integrity Server with FSC version 6.1-0 or later installed, one option is to clone the system drive to the backup drive as described in FES-03-101
    8. If the console is already running the FSC user application menu, it might already be at the Backup menu or still waiting for a typed response to complete a backup. In the case of the latter, respond appropriately to navigate to the Backup menu so that you can start a new backup.
      1. Note:

        If a reliable backup cannot be performed, resolve this problem before trying to run AUTOGEN.

      2. Note:

        If you do not have a password to access the Backup procedure and there is no one already assigned to make backups, use MOC to add a new user and password with the appropriate privilege for Backup.

    9. Upon making a successful image backup, exit from the FSC user account and start a new session with the SYSTEM username.
    10. At the command line prompt (a.k.a. DCL prompt), type the command:

      show system/sub

    11. Examine the first line reported to ensure that it has been more than seven days since the system was last rebooted. The following example shows a system that has been operating 32 days, 4 hours, 48 minutes, and 36 seconds (the Uptime).

      FSCISX, andgt;, andgt; show system/sub

    12. If the Uptime is sufficient, initiate the AUTOGEN procedure by typing the following command at the DCL prompt: @sys$ update:autogen savparams setparams
    13. Monitor all the messages to look for any that have “–E– “ or “–F– “ in the middle of the all capital letters part of the message. These are indications of problems which may require expert assistance to resolve. An example of a successful tuning execution, looks like the following log. The DCL prompt for this system is “FSCISX, andgt;, andgt;”.

      FSCISX, andgt;, andgt; @sys$update:autogen savparams setparams

      %AUTOGEN-I-BEGIN, SAVPARAMS phase is beginning.

      %AUTOGEN-I-FEEDBACK, Gathering feedback

      %DCL-I-IGNQUAL, qualifiers appearing before this item were ignored

      \SYMBOL\

      has been created. You may wish to purge this file.

      %AUTOGEN-I-END, SAVPARAMS phase has successfully completed.

      %AUTOGEN-I-BEGIN, GETDATA phase is beginning.

      %RENAME-I-RENAMED, SYS$SYSROOT:[SYSEXE]PARAMS.TMP;1 renamed to SYS$SYSROOT:[SYSEXE]PARAMS.DAT;6

      %AUTOGEN-I-NEWFILE, A new version of SYS$SYSTEM:PARAMS.DAT has been created.

      You may wish to purge this file.

      %AUTOGEN-I-END, GETDATA phase has successfully completed.

      %AUTOGEN-I-BEGIN, GENPARAMS phase is beginning.

      FSCISX, andgt;, andgt;

    14. If the tuning was successful, log off the session and coordinate with the users to schedule a system reboot.
    15. To reboot the system, first log onto the console terminal as user “fsc”. Then navigate to the Utilities menu and select the Shutdown/Reboot System option.

Required Training

Before performing this work instruction, applicable personnel must be trained by an authorized trainer on the required training contents listed in this document.

Required items

  1. Password to the SYSTEM account.
  2. Performance of an image backup immediately before an AUTOGEN is highly recommended.
  3. Experience with doing a backup recovery or ready access to an experienced person is highly recommended.

Definitions

FSC
Forming Supervising Computer – The server computer that connects to the Space Forming Machines
