Verifying Machine Server Backups

Purpose

This document describes how to verify that Machine Server backups are properly occurring. It assumes that SWSet 223 (SCS-0237-GR223) or greater is being used.

Application

This work instruction applies to all Space Plants that have FSC and Space IS Machines.

Safety

Follow all plant safety requirements.

Procedure

  1. On the backup Machine Server, log off the obop account (using the sequence Start-, andgt;Shutdown-, andgt;Log Off).
  2. Log onto the backup Machine Server using the administrator account (password: scsgolden).
  3. Right-click the My Computer icon, and select explore.
  4. Navigate to the C:\SpaceApps\Backups\, andlt;server name, andgt;directory, where , andlt;server name, andgt; is the name of the Machine Server backup to be verified.
  5. In this directory, confirm the presence of a file named backup.zip. It should have the current (today’s) date in the Date Modified column, with a time of approximately 1:00AM (see Figure 1).
  6. Double-click the backup.zip file.
  7. Open the temp directory in the backup.zip file (see Figure 2).
  8. Verify that the following files are present:
    1. eso1 (directory).
      1. Empty directory
  9. eso2 (directory).
    1. Empty directory.
  10. esov1 (directory).
    1. Profiles for the ESO.
  11. esov2 (directory).
    1. Empty directory.
  12. ldr1 (directory).
    1. Profiles for the lehr loader.
  13. ldr2 (directory).
    1. Empty directory
  14. ndl1 (directory).
    1. Profiles for the needles.
  15. ndl2 (directory).
    1. Empty directory.
  16. BackupFiles.bat.
  17. BAK.tkn.
  18. CS3.ini.
  19. CS3_1xx.DAT.
    1. The xx indicates the section’s IP address.
    2. These files exist for COM-SOC 3 lines only.
    3. These are the primary backup for the current running timing on the forming Machine. They should have a modified time between 12:40AM and 1AM of the current day.
  20. CS3_1xx.BAK.
    1. The xx indicates the section’s IP address.
    2. This files exist for OSI/EICOM lines only.
    3. These are the secondary backup for the current running timing on the forming Machine. They should have a modified time between 12:40AM and 1:00 AM of the current day.
  21. DEVICE.tkn.
  22. Machine_Configuration.txt
