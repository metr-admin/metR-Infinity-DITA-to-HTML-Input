To create a sourcing event

  1. Click the Sourcing Events tab.

    Or

    Click and select Sourcing Events.

  2. Click New.
  3. Enter the name of your sourcing event in the Event Name text box.
  4. Click the icon to select a Sourcing Event Type. From the lookup window, select the sourcing event type that you want to apply to your sourcing event.
    Note:

    Note

    Sourcing Category, Description, and Detailed Description are auto-filled on the basis of Sourcing Event Type you select.

  5. Complete the following fields.

    Field Name

    Description

    Estimated Value

    Enter the expected total value of the Sourcing Event.

    Sourcing Manager

    The system automatically populates the name of the user who has created the sourcing event. If you want to change the Sourcing Manager, click the icon and select an existing user.

    Note:

    The user to be selected for this role must have a profile of System Administrator.

    Project Manager

    Select the type of user from the drop down menu. You can select either a User or a Contact. Enter a name or look-up from the existing users for the role of Project Manager.

    Note:

    The user to be selected for this role must have a profile of System Administrator.

    Description

    Enter the description of the Sourcing Event.

    Detailed Description

    Enter the detailed description of the Sourcing Event.
  6. Click the icon for Sourcing Manager and select an existing user from the lookup window. The system automatically populates the name of the user who creates the sourcing event.
    Note:

    Note

    The users with a Salesforce license are displayed in the lookup window.

  7. Click the icon for Project Manager. From the lookup window, select a project manager for your sourcing event.
    Note:

    Note

    The users with a Salesforce license are displayed in the lookup window.

  8. Select the appropriate access for the sourcing event from the Sourcing Event Access picklist menu.

    Sourcing Event Access Type

    Description

    Public

    All the users having access to Sourcing Events object can view the Sourcing Event.

    Semi-Private

    Apart from the sourcing manager and the project manager, you can specify the users to whom you want to give access to the Sourcing Event.

    Private

    Only the sourcing manager and the project manager can view the Sourcing Event.
    Note:

    To set Sourcing Event Access for a Sourcing Event, refer to Setting Sourcing Event Access.

  9. Click Next.
  10. If you want your suppliers to sign an NDA agreement before they accept the invitation to the sourcing event, select NDA Required? check-box.
  11. Select the type of NDA from the NDA Type picklist menu. Available options are Master NDA and Event NDA.
    Note:

    Note

    To add a Master NDA for the supplier, refer to Adding Master NDA file.

  12. Click Edit to add the Event NDA file to the sourcing event.
  13. Click Choose File and select the appropriate file.
  14. Click Attach. The name of the file that you have uploaded is displayed in NDA File field.
    Note:

    Note

    You can not delete the NDA file once the Preparation phase is completed.

  15. Click to expand the Additional Info section. Enter a detailed description of your sourcing event.
  16. Click Upload to upload an attachment to the sourcing event.
    Note:

    Note

    For the Attachments in Apttus STP, you can upload any file. We allow files of the following extension types: (.PDF), (.RTF), (.RTX), (.TXT), (.JPG), (.JPEG), (.PNG), (.GIF), (.TIF), (.BMP), (.DOC), (.DOCX), (.DOCM), (.PPT), (.PPTX), (.PPTM), (.PPS), (.PPSX), (.XLS), (.XLSX), (.XLSM), (.ZIP), (.MP3), (.M4A), (.OGG), (.WAV), (.MP4), (.M4V), (.MOV), (.OGV), (.3GP), (.3G2). The application might become slow if you upload a file higher than 25 MB.

  17. Click Choose File and select the appropriate file.
  18. Click Open.
  19. Enter the File Description.
  20. Select the appropriate permission from the Permissions picklist menu.

    Permission

    Description

    Public

    To make the attachment available to all the Suppliers.

    Private

    To make the attachment available to only the Buyer.

    Selective Public

    To make the attachment available to the selected Supplier(s).

    Note: At least one supplier must be selected for this option
  21. Click Attach.
  22. To edit an attachment, click Edit.
  23. To delete an attachment, click Delete.
  24. Click Save.
  25. Click Activate Event.
  26. On the Phase Progression pop-up window, click Yes to activate the event. Or

    Click No to go back to the sourcing event.

Note:

Note

  • To save an event, click Save. Please see that, in order to save changes to the sourcing event, you must click Save. It is advisable to click Save at end of each phase. The instructions in this document are independent of other phases, hence, an instruction for saving the work is given keeping in mind a single activity. It is not mandatory to perform the Save action at end of each activity.
  • Click More to access the following actions:
    • To clone an event, click Clone Event.
    • To set an event on hold, click Set On-Hold.
    • To cancel an event, click Cancel Event.
