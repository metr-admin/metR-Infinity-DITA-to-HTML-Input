Managing Request for Proposal

Skip this section if your sourcing event is not of RFP type.

In Strategic Sourcing, Request for Proposal (RFP) facilitates RFI and RFQ simultaneously. You can run RFI and RFQ phases concurrently. RFI and RFQ works same as it does in the normal phase. The advantage of using RFP is it combines two phases (RFI and RFQ) and hence, they can be accessed independently and simultaneously. To enable this feature, Do you want to run RFI and RFQ concurrently? check box in your sourcing event type must be selected.

The chevrons will be created accordingly. The supplier will be notified via an email, if you have selected a template.

The key benefit of using RFP is it saves time. RFP makes evaluation simple as there are fewer documents to evaluate.

For more information on RFI , see RFI.

For more information on RFQ, see RFQ.

Note:

Note

Ensure that you click Start RFP when you want to start RFP phase. Click Complete RFP after your RFI and RFP are complete.
