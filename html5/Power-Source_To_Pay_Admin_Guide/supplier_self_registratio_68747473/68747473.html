Supplier Self-Registration

Supplier Self-Registration allows suppliers to have access to the 3rd party link to submit the Prospective Supplier Request Form. Onboarding surveys are also attached with the form to collect more details. The Supplier will fill the surveys and send the responses to you. You must create a supplier self-registration link and provide that link on your website or send the link via an email to the prospective suppliers.

Tip:

Use Case

Sam receives the Prospective Supplier Requests via an email. It may be a direct request or a reference. To reduce his work, Sam can provide the supplier self-registration link on the website of Tier 1 Mobiles or send this link via an email. In our case, following suppliers register themselves using this link.

To create a supplier self-registration link

You must have an existing community that you use for public RFI.

  1. Go to Setup > App Setup > Customize and select All Communities.
  2. Copy URL of the community that is used for public RFI.
  3. Append text /apex/Apttus_SRM__SupplierOnboardingRequest?type= to the copied URL.
  4. Append the Onboarding Type ID to the text obtained from the above step.
    Tip:

    To obtain the Onboarding Type ID,

    1. Click the Onboarding Types tab.

    2. Select the onboarding type that you want to assign to the supplier.

    3. Copy the onboarding type ID, text between = and &.

    For example, in https://c.cs26.visual.force.com/apex/OnboardingType?id=a5r2800000000kW&sfdc.override=1 URL, a5r2800000000kW is the onboarding type ID.

  5. At the end, your link will look like. 
    https://amttussuppliers-developer-edition.ap2.force.com/suppliers/apex/Apttus_SRM__SupplierOnboardingRequest?type=a5r2800000000kWAAQ
  6. Paste the entire link in the above step on your website or send this link to your suppliers via an email.

You will receive the supplier self-registration request when any supplier self-registers using this link that you provide.

After you receive the request, you must review and approve or reject the request. For more information on reviewing and approving or rejecting the request, refer to Supplier Review and Approvals.
