Post-Installation Configurations for May Release

You must perform the following settings after you install the May Release (Apttus STP version 7.44) packages in your org.

Warning:

Apttus recommends you to verify if you have performed the following post-installation settings in the last release. If they have not been performed, perform them in the current release. You do not need to configure them again if done.

Note:

Note

The following configurations are a part of Post-Installation Configuration. We are distinguishing the configurations pertaining to May release for the convenience of existing customers.

If you are an existing customer, go ahead and perform the configurations mentioned below. If you are a new customer, perform all configurations mentioned in Post-Installation Configurations.

  • Configure the existing System Administrator ProfileYou must configure the profile that you have assigned to the System Administrator. You must perform the following configurations.
    • Provide sObject Access

      You must provide the sObject Access to the objects for visibility to your users.

    • Provide Field-Level Security

      You must provide the field-level security to the objects for visibility to your users.

  • Configure the existing Suppliers Profile (Test Partner Community User)You must configure the profile that you have assigned to the suppliers. You must perform the following configurations.
    • Provide Field-Level SecurityYou must provide the field-level security to the objects for visibility to your users.
  • Configure the existing Guest User Supplier Profile (Suppliers Profile)You must configure the profile that you have assigned to the guest suppliers to access public community. You must perform the following configurations.
    • Enable VisualForce page to the community

      You must enable the visualforce pages to the community for visibility to your users.

    • Provide sObject Access

      You must provide the sObject Access to the objects for visibility to your users.

    • Provide Field-Level SecurityYou must provide the field-level security to the objects for visibility to your users.
  • Configure SRM System SettingsYou must configure the basic system settings that have been added. The following new settings have been added in May release:
    • Display Terms & Condition

      Selecting this check box will enable you to display your customized Terms and Conditions page on the Supplier Portal.

    • Terms & Condition Static Resource

      Enter the name of the static resource that you have created for uploading the file containing the terms and conditions.

    • Public RFI Welcome Message

      Enter the name of the static resource that you have created for uploading the file containing the terms and conditions.

    Warning:

    Warning

    You must configure SRM System Settings. The settings mentioned in SRM System Settings of May release (Apttus Supplier Relationship Management v7.44) are specific to May release only. If you are using any other version, please refer to the SRM System Settings pertaining to that particular release.

  • Enable Batch ProcessingYou must enable the batch processing for running the auto-progression of sourcing events.
  • Enable Survey Callback and Wizard Callback classes in Scorecard System SettingsYou must enter the Callback Class names in Scorecard System Settings.
  • Enable APIs in Salesforce OrgYou must enable APIs for the Community Supplier and Guest Supplier.

To enable batch processing

  1. Go to User Profile and select Developer Console.

  2. From Debug toolbar menu, select Open Execute Anonymous Window.
  3. Paste the following code in Enter Apex Code window. 
    datetime nextScheduleTime = system.now().addHours(1);
        string minute = string.valueof(nextScheduleTime.minute());
        string second = string.valueof(nextScheduleTime.second ());
        string cronvalue = second+' '+minute+' * * * ?' ;
        string jobName = 'AutoProgression ' + nextScheduleTime.format('HH:mm');
    
        //this will scheudle the next run
        Apttus_SRM.ScheduledBatchableAutoProgression scheduledBatchableAutoProgression = new Apttus_SRM.ScheduledBatchableAutoProgression();
        system.schedule(jobName, cronvalue , scheduledBatchableAutoProgression);
  4. Click Execute.
  5. Close the Developer Console.

To enable APIs in Salesforce Org

  1. Go to Setup > Administer > Manage Users > Profiles and select the profile that you have assigned to Supplier Portal Users. We have used Test Partner Community User.
  2. Click System Permissions. We have enabled the Enhance View of Profiles in User Interface for profiles.
  3. Click Edit.
  4. Select API Enabled check box.
  5. Click Save.
