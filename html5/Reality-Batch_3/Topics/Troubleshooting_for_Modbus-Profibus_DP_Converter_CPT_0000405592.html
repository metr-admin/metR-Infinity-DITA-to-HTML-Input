Troubleshooting for the LUFP7 Gateway

Issue

Check

Action

No LEDs are active on the LUFP7 gateway.
  • Check the health status of the DC supply
  • Check the polarity of the gateway DC supply

If the DC voltage supply is OK and the polarity is correct and no LED is active, the device needs to be checked by a maintenance engineer.

No communication on the Profibus bus.

That the RJ45 cable from the Ethernet patch panel is connected to the right port on the gateway.

Change the cable to the Modbus port on the gateway.

Incorrect gateway address.

Set the gateway address to the correct default value according to the table in chapter Default Profibus-DP Addresses.

No signal on the Modbus network, the LED (5) is red.

Check the addresses for the secondary devices on the Modbus RTU network.

Set the device Modbus address to the correct default value according to the tables in chapter Default Modbus RTU Addresses

Check that the wiring between the secondary Modbus devices and the Ethernet patch panel.

 Correct wiring according to the electrical drawings and the specifications in the chapter Modbus Wiring Between Devices.

Check the Modbus wiring between components (IM400, IFL12MC).

Check if any of the secondary Modbus devices (IM400, IFL12MC) are inoperative. If the Reality status LED is red on any device.

If the Reality status LED is red, the device needs to be checked by a maintenance engineer.

Check that the gateway is using the correct config file depending on solution type (PEW/SDC). For PEW solutions also check that the config file has the correct setup order (A060/A062/A064).

Replace the config file with the correct config file.

No communication with the config port of the LUFP7 gateway.

Check that the straight PowerSuite cable (VW3 A8 106) is connected to the config port and not the Modbus port.

Connect the straight PowerSuite cable (VW3 A8 106) to the config port of the gateway.

Check the correct COM port number.

Set the correct COM port number.
