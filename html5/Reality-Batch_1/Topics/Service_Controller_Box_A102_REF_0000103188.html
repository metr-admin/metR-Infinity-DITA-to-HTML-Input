Controller Box (A102)

Reality EMEA

The controller box controls the entire UPS system and consists of 2 PCBs:

  • Controller board

  • Adapter board

The controller box has the following functions:

  • controls the power modules

  • communicates with the Smart Slot Module (NMC, Modbus)

  • communicates with the HMI

  • communicates with the interface board

  • communicates with the external connection board via the interface board

  • communicates with the static bypass switch controller

Connections to the Controller Box

Number Connection Function
1 J201: A-J Communication ports to the other controllers, the HMI, the interface and the parallel board
2 D204: LED 1–4 Controller box status LEDs
3 J203: 1–96 96 pin connections to power module 2
4 J215: 1–2 24 SELV IN from power module 2
5 J226: 1–2 24 SELV OUT
6 J224: 1–8 Connection to the SmartSlot Board
7 J223: 1–9 Service port
8 J205: 1–3 Connection to the fan monitoring board to indicate if a fan becomes inoperable
9 J214: 1–2 24 SELV IN from power module 1
10 J225: 1–2 24 SELV OUT
11 J202: 1–96 96 pin connections to power module 1

Adapter Board

The adapter board is integrated in the controller box. It adapts the interfaces of the SUVT controller board to the mechanical build-up of the system.

Functions of the adapter board:

  • All input/output signals of the controller box are routed via the adapter board. All connectors of the controller box (except the one to the power connection board) are directly mounted to the adapter board.

  • Decoupling of redundant supplies

  • Four status LEDs

  • Controller monitoring and selection of local master controller

  • Galvanic isolation between ELV and SELV parts

  • Measurement of bypass voltage via the measurement resistors on the power connection board

  • Driver circuits to use additional FPGA output signals

  • Routing of signals from the power module connectors to the controller board

  • Charge current symmetry for the battery charger

Controller Box Status LEDs

Four LEDs on the controller box indicate the status:
LED Status Description
LED 1: Green Controller OK –
LED 2: Yellow Local master In all individual systems, this LED will be illuminated in the system with controller number 1, and will be off in all other systems
LED 3: Yellow System isolated In parallel systems: The parallel systems are not connectedIn single systems: This LED is never illuminated
LED 4: Yellow Global master In parallel systems: This system is masterIn single systems: This is the same as the local master
