Static Bypass Switch Controller Board (A401)

Reality EMEA

The static bypass switch has its own controller. If the output voltage goes out of tolerance due to a malfunction in the main controller, the static bypass switch controller becomes active and independent. Following a malfunction in the main controller, the static bypass switch controller takes over and transfers immediately to static bypass.

Under normal operating conditions, the static bypass switch is fully controlled by the UPS controller. The static bypass switch controller analyses the control signals from the UPS controller, performs the corresponding tasks, and returns a status signal to the UPS controller. If the communication fails, the static bypass switch can operate independently.

The firmware mainly performs the following tasks:

  • Measures the control frequency coming from the UPS controller and reacts accordingly

  • Controls the thyristors

  • Generates a status feedback signal for the UPS controller

  • Monitors the PSU

  • Controls three LEDs and two open collector outputs to display the actual state

  • Measures and evaluates the system output voltage

ARM State (Normal Operation)

When the control signal is lost in ARM state, the output voltage is monitored by the fast-detector during 500 ms:

  • If the output voltage remains OK, the static bypass switch switches to DISARM after 500 ms.

  • If the output voltage is lost within these 500 ms, the static bypass switch switches to FIRE state.

After the static bypass switch has switched to FIRE state, it again monitors the output voltage (RMS detector and DC detector):

  • If the voltage is lost, the static bypass switch switches to ERROR state. A reset is then required in order to turn on the static bypass switch again. Within the first 200 ms after switching to FIRE, the static bypass switch does not switch off.

  • If DC is detected on the output, the static bypass switch switches to ERROR state. A reset is then required in order to turn on the static bypass switch again. Within the first 200 ms after switching to FIRE, the static bypass switch does not switch off.

FIRE State (Requested or Temporary Static Bypass Operation)

When the control signal is lost in the FIRE state, the static bypass switch stays in the FIRE state. The static bypass switch monitors the output voltage (RMS detector and DC detector):

  • If the voltage is lost, the static bypass switch switches to ERROR state. A reset is then required in order to turn on the static bypass switch again.

  • If DC is detected on the output, the static bypass switch switches to ERROR state. A reset is then required in order to turn on the static bypass switch again.

Regaining Control over the Static Bypass Switch

When the UPS controller recovers from an inoperable state or a cable is reconnected, control over the static bypass switch can be regained.

  • If the static bypass switch is independently in FIRE state, control cannot be regained. A firmware restart (reset) is required.

  • If the static bypass switch is in ERROR state due to independent switching off from FIRE state (low voltage or DC detection), control cannot be regained. A firmware restart (reset) is required.

  • If the static bypass switch is in another state, it reacts immediately to the regained control signal.
