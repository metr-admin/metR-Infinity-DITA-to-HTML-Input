Functioning of the LCM

Reality EMEA

LCM at System Start-Up

During system start-up, the LCM module checks if the counters have already been started.

If the counters have been started before the actual start-up date, the actual LCM status is reloaded from the Flash memory.

If the counters have not been started before the start-up, the actual date is stored in RAM and will be used as the system start-up date.

  • Once the system has been running (at least in standby) for 5 consecutive days without interruption, the LCM module considers the system to be commissioned and automatically stores the initial start-up date in the Flash memory as the UPS start-up date. This date is also stored as the start date for each of the five LCM counters.

  • If during these first five days, the LCM action Start counters now is performed by the FSR, the actual date is stored as the UPS start-up date and as start date for each of the five LCM counters.

LCM During Runtime

After the counters have been started, the LCM module checks once per day if a counter has reached its elapse date. If the elapse date has been reached, the corresponding counter elapse flags this and the number of remaining pop-up repeats are set in RAM and Flash. An entry is added to the event log.

Once per minute, the LCM module checks if there is a pop-up message to be displayed on the display. A pop-up will be displayed if the elapse flag of a counter is set, there are remaining repeats, and the repeat period is over. If another LCM pop-up is already shown on the display, this will not be overwritten.
