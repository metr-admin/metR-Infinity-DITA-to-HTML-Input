Life Cycle Monitoring (LCM)

Reality EMEA

The standard Life Cycle Monitoring (LCM) feature uses basic time counters for the monitored components. For each component an initial expected lifetime is configured at first start-up of the product, and this lifetime will be decreased by the same value day by day until it reaches zero.

The actual operating conditions of the component will have no impact on the estimated remaining lifetime, only the time in operation has. This means that the estimated lifetime will not be accurate, except if the component is only exposed to the wear and exact conditions taken into account when the initial nominal lifetime was calculated.

Life Cycle Reminders

There are five possible reminders.
# Reminder
1 Start-up reminder
2 Fan module replacement recommended
3 Battery replacement recommended
4 Power module replacement recommended
5 Air filter replacement recommended
6 Warranty expired
After a configured period (individual periods for each reminder), an event pop-up is shown on the display and the event is logged in the event log.

An LCM reminder can be temporarily removed from the display by the user by pressing the Enter key to acknowledge the alarm, but after a defined period (default is set to 30 days) the pop-up message will reappear. After a configured amount of repeats (default is set to 3 times), the pop-up will not reoccur. Final acknowledgement and timer restart can only be performed by an FSR.

The Common Alarm LED is blinking (0.5s ON, 0.5s OFF...) and the buzzer is activated when an LCM reminder pop-up message is shown on the display.

Setting Sheet

All configuration parameters of the LCM functionality are stored in the setting sheet. The parameters are set to default values and normally require no modification during system configuration.

The LCM module also stores its counters’ status information in a setting. This value requires no modification.

To modify the LCM configuration parameters, see Service Port LCM Status Menu and Service Port LCM Action Menu.
