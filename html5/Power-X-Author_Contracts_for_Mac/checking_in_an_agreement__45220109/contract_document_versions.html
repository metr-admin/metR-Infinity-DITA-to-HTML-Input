Contract Document Versions

When you check in documents from X-Author, document versions are incremented based on Contract Document Versioning. For example, if you work with version 1.0.0 (the first generated draft of an agreement), and check it in as a new redlined version, the version of the document increments to a new "negotiator" version, version 1.1.0. The following section describes how Contract Document Versioning works. Keep in mind that minor (reviewer) versions cannot be created from the X-Author client but that you may work potentially check out reviewer versions as part of the negotiation process.
